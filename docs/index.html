"It runs" is not "it's safe"

When you don't know what you're doing, an AI tool still gets you to working software. The part it doesn't tell you is everything underneath. These are the questions Vibe Check is built to answer honestly.

Did I leave a security hole? Did I hardcode a secret? Will it hold at ten times the users? Can I keep building on it without it turning to mud?
How it works
01

Point it at your app

Drop the folder next to a project you shipped and open it in your own Claude Code. No account, no upload, no service.

02

It reads, never runs

It reviews the code the way a senior engineer would: auth, access control, data handling, dangerous shortcuts, and the test surface. Your code stays on your machine.

03

You get the truth

An honest Build Review Report: a clear verdict, the fixes worst first, and the list of what a read of the code cannot prove.

[ ! ]

The one rule it can't break

It never calls your build "secure" or "ready" while anything serious is still open, and it always shows the full list of what it could not check. A small script, check.py, enforces that before any report reaches you. The reviewer cannot talk its way past it, and neither can a hidden instruction buried in your code.

What you get back
[=]

A clear go or no-go

One honest verdict at the top, not a wall of maybes you have to decode.

[#]

A fix list, worst first

Each item in plain English with the exact change to make, ordered by how much it matters.

[?]

An honest could-not-check list

The things a read of the code cannot settle, named in full, so nothing looks covered that isn't.

See it bite, then see it stay honest

First, the teeth. Pointed at a demo app built to fail on purpose, it returns a flat NO-GO and names the real problems in plain English: a login an attacker can walk straight through, an admin page that hands out passwords with no sign-in, and a secret key typed into the source. Every one comes with the exact fix. No hand-waving, no "looks mostly fine."
